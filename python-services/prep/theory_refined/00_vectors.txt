   ChromaDB's role — it would have handled both storage and search (it
  stores vectors + runs HNSW similarity search). The code explicitly
  explains why it was dropped (retriever.py:12-16, numpy_index.py:7-9):

  ▎ ChromaDB runs in a separate container — every query was a network
  ▎ roundtrip (~50–150ms). NumpyIndex does the same cosine similarity in
  ▎ ~0.05ms in-process with a single matrix-vector multiply.
