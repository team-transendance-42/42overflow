Two completely different layers:

  Browser (Svelte)
        │
        │  POST /ai-assist/ollama        ← our route, on our Go server :8081
        ▼
  Go Backend (llm-system-interface)
        │
        │  POST http://ollama:11434/api/chat   ← OLLAMA'S route, on Ollama's local server
        ▼
  Ollama (local Docker container)

  ---
  Why /api/chat?

  /api/chat is Ollama's own fixed REST endpoint — it's baked into the Ollama binary, not something we chose. we're calling Ollama the same way
  we'd call any local HTTP service. "Local" just means it runs in a Docker container on our network, not that it has a special protocol — it
  still speaks HTTP.
  our routes (/ai-assist/ollama, /ai-assist/gemini, /ai-assist/community) are what the browser talks to. They're our API surface, our naming.
  ---
  Full pipeline per request type:

  Browser
    └─▶ Caddy :8443  (HTTPS, only public port)
          └─▶ SvelteKit :5173  (auth check via locals.user)
                ├─▶ /api/ollama     → Go :8081/ai-assist/ollama     → Ollama :11434/api/chat
                ├─▶ /api/ai-assist  → Go :8081/ai-assist/gemini     → Gemini API (external)
                ├─▶ /api/community  → Go :8081/ai-assist/community  → python-rag :8090/rag/retrieve
                │                                                       └─▶ ChromaDB :8000
                │                                                       └─▶ Postgres :5432
                │                                                    → Ollama :11434/api/chat
                └─▶ /api/stt        → python-stt :8091/convert_audio  (Whisper)
